# design-system（全社共通デザインの置き場）

全プロダクト（交通誘導警備＝Esecurity／人材事業＝Esupport ほか）の**見た目・作法を統一**するための共通置き場。**ここが唯一の正（source of truth）**。

## 入っているもの
- **`tokens.css`** … 公式カラートークン17色（確定・不変）。色の正。
- **`RULEBOOK.md`** … 全社共通の統一ルール（色・操作の作法・用語・入力形式・ログイン・共通部品の扱い 等）。
- （今後）**共通部品**（ボタン・表・入力・モーダル・ランチャー 等）を、必要になった都度ここへ追加（“昇格”）。

## 運用ルール（ガバナンス）
- **変更できるのは社長のみ。** 各製品の担当者は**「使うだけ」**で、ここを書き換えない。
- 「共通部品になりそう」と気づいたら、各自で共通化せず**社長に相談** → 社長が判断してここへ昇格 → 配布。
- 直書きの色は禁止。各製品は `tokens.css` の `var(--token)` を使う。

## 使い方（段階）
- **今（モック段階）**：各製品は本リポの `tokens.css` を参照（取り込み）し、`RULEBOOK.md` に従って作る。各製品の `CLAUDE.md` にも同ルールを記載済み。
- **本実装段階（React等）になったら**：本リポを**パッケージとして配布**し、各製品は“インストールして使うだけ”に切り替える（読むだけ消費）。共通部品もここから配布。

## メモ
- 個人アカウントの非公開リポは「読むだけ招待」が不可（Organizationのみ）。従業員への“読むだけ”は、本実装段階のパッケージ配布で担保する。それまでは RULEBOOK＋各製品のCLAUDE.md＋（後で）CIで統一を守る。
