# 標準エージェント（サブエージェント）定義

ここに置く `*.md` が「全社標準のサブエージェント」です。
`setup.sh` でこのフォルダの内容を各自の `~/.claude/agents/` に取り込みます。
＝全員が同じ専門エージェントを使える状態になります。

## 今ある標準エージェント
| 名前 | 役割 | いつ使う |
|---|---|---|
| `design-guard` | 全社共通ルールの番人レビュー（色/アイコン/フォント/サイドメニュー・ヘッダー/骨格一致を点検） | コミット・PRの前、「ルールに合ってる？」と確認したい時 |

## 追加・変更のルール
- 標準エージェントを足す/変えるのは **社長の承認（PR）**でのみ。
- 各自が勝手にローカルで別エージェントを増やしても、最終的な成果物は各製品のCI（番人）で
  チェックされるため、ルール違反はマージできません。標準をそろえる目的は「入口の統一」です。

## 形式（参考）
サブエージェント定義は、先頭にYAML（`name` / `description` / 任意で `tools`）を書き、
その下に役割の説明（システムプロンプト）を書いた Markdown です。`design-guard.md` を雛形にしてください。
