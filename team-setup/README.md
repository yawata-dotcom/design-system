# 共通の道具セット（team-setup）

全社員・共同開発者が **同じ作業環境（AIエージェント・スキル・作業ルール・推奨設定）** で
開発を始めるための「1か所にまとまった道具セット」です。
ここを見れば、社長（管理者）と同じ道具がそろいます。

> ねらい：誰がどのAIで作っても、**見た目と品質がそろう**こと。
> 実際の最終保証は各製品のCI（番人）＝色・アイコン・フォント・サイドメニュー/ヘッダーの
> 自動チェックでかかっています（道具に関係なく成果物で弾く）。ここはその「入口の統一」です。

---

## 入っているもの

| 中身 | ファイル | 説明 |
|---|---|---|
| ① 作業ルール | `rules/CLAUDE.shared.md` | 全社共通の進め方・絶対ルール（公開RULEBOOKを正として参照） |
| ② エージェント設定 | `agents/` | 標準のサブエージェント定義（例：`design-guard` = ルール番人レビュー） |
| ③ スキル一覧 | `skills.manifest.md` | 使う11スキルの名前・用途・入手元・ライセンス（実体は同梱しない＝下記参照） |
| ④ 推奨設定 | `settings.recommended.json` | 許可する道具・外部連携(MCP)の標準（セキュリティ） |
| ⑤ 導入スクリプト | `setup.sh` | ①②④を自分の環境へ取り込み、③の導入手順を表示 |
| ⑥ 製品台帳 | `products.md` | 全プロダクトと「強制状況」の一覧（＝強制の外にいる製品を見える化・漏れ防止） |
| ⑦ 新リポ一発セットアップ | `bootstrap-repo.sh` | 新プロダクトに番人＋CIひな形を配備し、保護設定手順を表示 |
| ⑧ 並行作業の手順 | `parallel-work.md` | 2台のPCで作業がかぶらず安全に進める方法（同一プロダクト分担／PC別プロダクト 両対応） |

> **新しいプロダクトを作る時は必ず `products.md` の手順に従い、強制下に置く**（番人＋ブランチ保護）。
> ロックはPCではなく「GitHubのリポ単位」で効くため、新リポは作った瞬間に仕組みを入れないと強制の外になります。

### スキルを同梱しない理由
スキルは配布元のライセンスに配慮し、**この公開リポには実体を入れません**。
代わりに `skills.manifest.md` に「名前・入手元」を載せ、各自が同じ公開元から入れます。

## 使い方（導入）

```sh
# 1) このリポを取得（または最新化）
git clone https://github.com/yawata-dotcom/design-system.git
cd design-system

# 2) 道具セットを自分の環境へ取り込む（バックアップを取ってから反映・安全設計）
sh team-setup/setup.sh

# 3) 画面に出る案内に従って skills.manifest.md のスキルを入れる
```

`setup.sh` は **既存ファイルを上書きする前に必ずバックアップ** を作ります。
推奨設定は勝手に丸ごと置き換えず、内容を表示して取り込み方を案内します。

---

## 変更のしかた（重要）
道具セットの中身を変えたい時は、**この `team-setup/` を直して PR**（社長承認）で反映します。
＝道具の標準も、サイドメニュー/ヘッダーと同じく「社長が承認したものだけ」が正になります。
各自は次回 `git pull` ＋ `setup.sh` で最新の標準に追従します。
